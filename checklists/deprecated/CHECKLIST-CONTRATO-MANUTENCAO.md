# CHECKLIST – CONTRATO DE MANUTENÇÃO / CORREÇÃO CONTROLADA

Checklist de validação de correções sob contrato de manutenção.

---

## 1. Natureza

- [ ] Solicitação explicitamente marcada como manutenção
- [ ] Nenhum RF ou execução de desenvolvimento envolvida

---

## 2. Escopo

- [ ] Correção limitada ao erro descrito
- [ ] Nenhuma funcionalidade nova criada
- [ ] Nenhuma refatoração ampla

---

## 3. Correção

- [ ] Causa raiz identificada
- [ ] Correção aplicada apenas no ponto necessário
- [ ] Nenhum efeito colateral detectado

---

## 4. Validação

- [ ] Erro não ocorre mais
- [ ] Sistema funciona após restart
- [ ] Testes existentes continuam passando
- [ ] Teste de regressão criado ou ajustado (se aplicável)
